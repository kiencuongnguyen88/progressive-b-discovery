# Human Feedback Template

Human feedback is the main signal that helps Progressive B Discovery move forward.

The human does not need to understand every technical detail.

The human needs to describe what happens in real use: what works, what does not fit, what feels unclear, what feels useful, and what should change in the next loop.

Vietnamese name: **Mẫu phản hồi Human cho Điểm B phát triển**

---

## 1. Why Human Feedback matters

In Progressive B Discovery, the final target B is often unclear at the beginning.

AI can build fast, but speed alone does not tell us whether the product is moving in the right direction.

Real use creates the most important signal.

Human feedback helps answer:

- Is this useful in reality?
- What part fits the workflow?
- What part feels wrong?
- What is missing?
- What is excessive?
- What is confusing?
- What is not trusted yet?
- What should be patched?
- What should be rebuilt?

Short principle:

    Human gives reality feedback.
    AI uses that feedback to make B clearer.

---

## 2. When to use this template

Use this template after:

- a rough version has been created;
- a tool has been used in a real workflow;
- a document or method draft has been reviewed;
- an AI-generated output feels close but not yet right;
- the human sees what should be kept or changed;
- the next step may be patch, checkpoint, or rebuild.

This template is useful even when the feedback is rough.

Rough feedback is better than silent confusion.

## 23. Example Feedback

### Feedback ID

    PBD-FB-001

### Project / Tool / Method Name

    Progressive B Discovery public repo

### Version Reviewed

    README.md v0.1

### Current B

    A public method repo that helps humans and AI agents build clearer products through runtime loops, feedback, lineage, checkpoints, and rebuilds.

### First Reaction

    Useful and directionally right, but the README is long.

### What Works

- The main loop is clear.
- The 3D print analogy is useful.
- The method is understandable for non-developers.
- The checkpoint idea gives the project a recovery path.

### What Does Not Fit Yet

- The README contains too many templates.
- Some sections should become separate files.

### Missing

- A quick-start guide.
- A short example project.
- A simple prompt for asking AI to run the loop.

### Excessive

- Too much explanation inside README.

### Not Trusted Yet

- It needs a real example to prove that the method works for outside readers.

### Human Decision

    Split templates into separate files, then create a short quick-start guide.

### Instruction for AI

    Continue from this feedback.
    Keep the main concept.
    Split reusable templates into separate files.
    Create a short quick-start guide after templates are separated.
